How Docker Works

Docker is a containerization platform that packages applications and their dependencies into lightweight, portable containers. Unlike virtual machines, containers share the host OS kernel, making them more efficient.

Key Concept: Docker containers are isolated processes that run on a shared kernel, providing consistency across different environments while maintaining resource efficiency.

How Docker Compose Works

Docker Compose is a tool for defining and running multi-container Docker applications. It uses a YAML file to configure application services, networks, and volumes.

# docker-compose.yml example version: '3.8' services: nginx: build: ./requirements/nginx ports: - "443:443" networks: - inception networks: inception: driver: bridge

‚ö°Docker vs Virtual Machines

Docker Advantages

Docker containers offer significant benefits over traditional virtual machines:

  • Resource Efficiency: Containers share the host OS kernel, using less memory and CPU
  • Faster Startup: Boot in seconds vs minutes for VMs
  • Smaller Size: Images are typically MB vs GB for VM images
  • Better Portability: Run consistently across different environments
  • Microservices Ready: Perfect for modern application architectures
  • DevOps Integration: Seamless CI/CD pipeline integration

üîíNGINX SSL/TLS Configuration

SSL/TLS Requirements

The project requires NGINX to serve content exclusively over HTTPS (port 443) with proper SSL/TLS configuration.

# nginx.conf server { listen 443 ssl http2; server_name login.42.fr; ssl_certificate /etc/nginx/ssl/inception.crt; ssl_certificate_key /etc/nginx/ssl/inception.key; ssl_protocols TLSv1.2 TLSv1.3; location / { proxy_pass http://wordpress:9000; } }

SSL/TLS Checklist

  • Port 443 only (no port 80 access)
  • Self-signed certificate is acceptable
  • TLS v1.2/v1.3 support required
  • Proper certificate file paths
  • HTTPS redirection configured

üìùWordPress with PHP-FPM Setup

Key Requirements

WordPress must run with PHP-FPM and have persistent data storage through Docker volumes.

  • No NGINX in WordPress Dockerfile
  • PHP-FPM for processing PHP requests
  • Volume Mount: /home/login/data/wordpress
  • Admin User: Username cannot contain 'admin' or 'Admin'
  • User Management: Support for comments and page editing
# WordPress Dockerfile example FROM alpine:3.17 RUN apk add --no-cache php81 php81-fpm php81-mysqli \ php81-json php81-curl php81-dom php81-exif \ php81-fileinfo php81-mbstring php81-openssl \ php81-xml php81-zip wget COPY conf/www.conf /etc/php81/php-fpm.d/ COPY tools/setup.sh /usr/local/bin/ ENTRYPOINT ["sh", "/usr/local/bin/setup.sh"]

üóÑÔ∏èMariaDB Database Setup

Database Requirements

MariaDB serves as the backend database for WordPress with persistent storage and proper configuration.

  • Custom Dockerfile (no pre-built images)
  • No NGINX in the Dockerfile
  • Volume mount: /home/login/data/mariadb
  • Database must not be empty
  • Proper user authentication setup
  • WordPress database configuration
# MariaDB initialization CREATE DATABASE wordpress; CREATE USER 'wpuser'@'%' IDENTIFIED BY 'password'; GRANT ALL PRIVILEGES ON wordpress.* TO 'wpuser'@'%'; FLUSH PRIVILEGES;
